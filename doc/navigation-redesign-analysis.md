# 导航栏设计分析与改进方案

## 一、现状分析

### 当前导航结构

| 导航项 | 类型 | 行为 |
|-------|------|------|
| 限时活动 | 页面跳转 | `goToPromotion()` → `/zh-cn/promotion` |
| 产品功能 | **首页内链** | `goSection('features')` → 滚动到 `#features` |
| 安全保障 | **首页内链** | `goSection('security')` → 滚动到 `#security` |
| 合伙人 | 页面跳转 | `goToPartner()` → `/zh-cn/partner` |
| 关于我们 | 页面跳转 | `goToAboutUs()` → `/zh-cn/about-us` |
| 联系我们 | **首页内链** | `goSection('contact')` → 滚动到 `#contact` |

### 问题总结

> [!WARNING]
> **核心问题：内链与页面跳转混杂，用户认知混乱**

1.  **混合导航类型**：
    - "产品功能"、"安全保障"、"联系我们" 是首页的锚点链接（内链）
    - "限时活动"、"合伙人"、"关于我们" 是独立页面
    - 用户无法从视觉上区分两者，点击后行为不一致

2.  **跨页内链体验差**：
    - 当用户在 `/about-us` 页面点击"产品功能"，会先跳转到首页再滚动
    - 这种"先跳转后滚动"的体验不够流畅
    - `goSection` 函数在非首页时会调用 `router.push` 后立即 `scrollIntoView`，可能存在时序问题

3.  **信息架构不清晰**：
    - "联系我们"作为一个首页 section，却和"关于我们"（独立页面）并列
    - 用户期望"联系我们"是一个完整页面，但实际只是滚动到首页底部

---

## 二、改进方案

### 方案 A：统一为独立页面（推荐）

> **思路**：将所有导航项都改为独立页面，彻底消除内链

**变更：**
| 导航项 | 新行为 |
|-------|--------|
| 限时活动 | `/promotion` (保持) |
| 产品功能 | `/features` **(新建页面)** |
| 安全保障 | `/security` **(新建页面)** |
| 合伙人 | `/partner` (保持) |
| 关于我们 | `/about-us` (保持) |
| 联系我们 | `/contact` **(新建页面)** |

**优点：**
- 导航行为一致，所有点击都是页面跳转
- 符合用户心理预期
- 每个页面可以有更丰富的内容和 SEO 优化

**缺点：**
- 需要新建 3 个页面（Features、Security、Contact）
- 首页内容需要调整

---

### 方案 B：保留内链但视觉区分

> **思路**：保持现有结构，但通过视觉设计区分内链和页面

**变更：**
1.  **分组显示**：
    ```
    [限时活动] | [产品功能 ▼] | [合伙人] | [关于我们] | [联系我们 ↓]
                  └─ 安全保障
    ```
    - 将"产品功能"和"安全保障"合并为一个下拉菜单
    - 或者在导航项后添加 `↓` 图标标识内链

2.  **样式区分**：
    - 页面链接：正常样式
    - 内链：带有下划线或不同颜色

3.  **代码优化**：
    - 改进 `goSection` 函数，使用 `router.push` 的 `hash` 参数
    - 在首页 `onMounted` 中监听 hash 变化并滚动

**优点：**
- 改动较小
- 不需要新建页面

**缺点：**
- 用户仍需学习两种导航模式
- "联系我们"作为内链可能不符合用户预期

---

### 方案 C：精简导航 + Footer 补充

> **思路**：主导航只保留核心页面，将内链移至 Footer 或首页 Hero

**变更：**
- **主导航**：限时活动 | 合伙人 | 关于我们
- **首页 Hero**：快速跳转按钮（产品功能、安全保障、联系我们）
- **Footer**：完整站点地图，包含所有链接

**优点：**
- 导航更简洁
- 内链逻辑清晰（只在首页出现）

**缺点：**
- 可能降低"产品功能"和"安全保障"的曝光度

---

## 三、推荐方案

> [!IMPORTANT]
> **推荐采用方案 A（统一为独立页面）**

### 理由：
1.  **一致性**：所有导航行为相同，降低用户认知成本
2.  **可扩展性**：每个页面可以独立优化 SEO
3.  **专业性**：符合高端金融科技产品的设计标准

### 实施步骤：
1.  **创建新页面**：
    - `FeaturesPage.vue` - 产品功能详情
    - `SecurityPage.vue` - 安全保障详情
    - `ContactPage.vue` - 联系方式（可包含表单）

2.  **更新路由**：
    ```typescript
    { path: '/:lang/features', component: FeaturesPage },
    { path: '/:lang/security', component: SecurityPage },
    { path: '/:lang/contact', component: ContactPage },
    ```

3.  **更新导航**：
    - 将 `goSection` 调用替换为页面跳转
    - 移除首页特殊处理逻辑

4.  **调整首页**：
    - 保留 Features/Security/Contact 区块作为"预览"
    - 添加"了解更多"按钮跳转到详情页

## 四、快速修复（临时方案）

如果暂时不想创建新页面，可以先做以下调整：

1.  **移除"联系我们"**：将其从导航栏移除，仅在 Footer 展示
2.  **合并内链**：将"产品功能"、"安全保障"合并为一个下拉菜单"探索 ▼"
3.  **优化 goSection**：使用 `router.push({ path: '/', hash: '#features' })` 替代当前逻辑

```typescript
// 改进后的 goSection
const goSection = async (sectionId: string) => {
  await router.push({ 
    path: `/${locale.value}`,
    hash: `#${sectionId}`
  });
};
```

然后在 `WebLayout.vue` 或首页的 `onMounted` 中添加：

```typescript
watch(() => route.hash, (hash) => {
  if (hash) {
    const el = document.querySelector(hash);
    if (el) el.scrollIntoView({ behavior: 'smooth' });
  }
}, { immediate: true });
```
